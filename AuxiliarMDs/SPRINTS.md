# Sprints Documentados

## Sprint 1 — Diseño y datos iniciales
- Definición del problema y alcance.
- Estructura del repositorio y dependencias.
- Generador de datos sintéticos y esquema CSV.
- Primer borrador de `README` y plan documental.

## Sprint 2 — Pipeline ETL y ML
- Implementación de carga y limpieza de datos.
- Entrenamiento de modelo con `scikit-learn`.
- Implementación de API con FastAPI y rutas de predicción.
- Tests unitarios iniciales.

## Sprint 3 — Automatización, Docker y presentación
- Dockerfile y `docker-compose` para API y dashboard.
- GitHub Actions para tests y build.
- Documentación de arquitectura y presentación.
- Creación de memoria del proyecto y backlog.
