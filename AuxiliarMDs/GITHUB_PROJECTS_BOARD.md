# GitHub Projects Board

## Tablero sugerido
- Columnas: Backlog, En progreso, En revisión, Completado.
- Etiquetas: ETL, ML, API, Dashboard, Infraestructura, Documentación.

## Tarjetas clave
- Crear generador de datos sintéticos.
- Implementar ETL y cargar datos procesados.
- Entrenar modelo ML y guardar el artefacto.
- Exponer API REST de predicción.
- Desarrollar dashboard con Streamlit.
- Añadir Dockerfile y docker-compose.
- Configurar GitHub Actions para tests y build.
- Escribir memoria del proyecto y presentación.

## Cómo usarlo
1. Crear proyecto en GitHub Projects.
2. Añadir tarjetas según el backlog.
3. Mover tareas entre columnas a medida que avanzan.
4. Documentar los sprints y la retrospectiva en `AuxiliarMDs/`.
